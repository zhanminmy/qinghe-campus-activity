一、项目概述

1.1 项目背景

校园活动是学生拓展兴趣、锻炼能力和建立社会联系的重要途径。实际校园生活中，活动信息常分散在不同通知群、海报或公众号中，学生难以及时获取完整信息。为改善这一问题，本项目设计并实现“青禾校园活动站”，集中展示校园讲座、公益、文体和创新创业活动，并提供分类浏览、详情查看和报名意向登记等功能。

1.2 建设目标

项目目标是完成一个结构合理、可以正常访问、具备一定交互特效且底层代码清晰的校园活动网站。网站同时兼顾电脑和手机端显示效果，通过统一的视觉规范和模块化代码，提高页面的可读性、可维护性与使用体验。

1.3 开发技术

技术用途
HTML5构建页面语义结构，包括导航、主体、表单和页脚。
CSS3完成网格布局、响应式适配、主题配色、过渡和动画效果。
JavaScript实现轮播、筛选、主题切换、弹窗、表单校验等交互逻辑。
LocalStorage在用户设备上保存深色或浅色主题偏好。

本项目采用结构、表现和行为分离的组织方式。HTML 文件负责内容与语义结构，CSS 文件负责全站视觉和响应式适配，JavaScript 文件集中管理公共交互及活动数据，避免在页面中重复编写逻辑。

3.3 视觉设计

网站以自然绿色和活力橙色作为主色，绿色体现成长、连接和校园氛围，橙色用于突出按钮和重要信息。页面使用卡片、留白和圆角建立清晰的信息层级，并提供深色主题以适应不同浏览环境。

四、关键功能实现

4.1 轮播与滚动动画

首页轮播通过数组索引控制当前显示内容，支持自动播放、上一张、下一张及圆点切换。滚动渐入使用 IntersectionObserver 监听页面元素进入可视区域，在满足条件时添加样式类，从而降低不必要的滚动计算。

4.2 活动分类筛选

每个活动元素使用 data-category 属性标记类别。当用户点击筛选按钮时，脚本对类别进行比较，显示符合条件的活动，同时更新页面中的结果数量。该方案代码简洁，增加新类别时也容易扩展。

4.3 动态详情与表单验证

活动详情页读取网址中的 id 参数，并从 JavaScript 数据对象中加载对应内容，实现一个页面展示多项活动。报名和留言表单会检查必填项、邮箱格式及手机号码格式，验证通过后显示成功反馈。该项目为前端演示，不会向服务器上传真实信息。

六、项目总结

通过本次实践，我完成了从需求分析、页面规划、视觉设计、代码实现到功能测试的完整网站开发过程。项目实现了多页面导航、响应式布局和多种交互功能，也让我进一步理解了 HTML、CSS 与 JavaScript 的职责划分。开发过程中，我重点关注代码复用和结构清晰，将公共样式与交互逻辑集中管理，并通过语义标签、键盘操作支持和减少动画偏好适配提升网站可访问性。

目前项目使用静态数据，适合作为前端网站实践作品。后续可以增加数据库和用户登录功能，将报名信息保存至后台，并加入管理端活动发布、报名审核和数据统计等模块，使网站形成完整的校园活动服务系统。
